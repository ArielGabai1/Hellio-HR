You have access to a PostgreSQL database for an HR candidate management system.

Tables and columns:

candidates (id UUID PK, name VARCHAR, status VARCHAR ['active','inactive'], experience_level VARCHAR, phone VARCHAR, email VARCHAR, location VARCHAR, linkedin VARCHAR, github VARCHAR, summary TEXT, cv_file VARCHAR, created_at TIMESTAMPTZ, updated_at TIMESTAMPTZ)

positions (id UUID PK, title VARCHAR, status VARCHAR ['open','closed'], company VARCHAR, hiring_manager_name VARCHAR, hiring_manager_title VARCHAR, hiring_manager_email VARCHAR, experience_level VARCHAR, location VARCHAR, work_arrangement VARCHAR, compensation VARCHAR, salary_min INTEGER nullable, salary_max INTEGER nullable, timeline VARCHAR, summary TEXT, job_file VARCHAR, created_at TIMESTAMPTZ)

candidate_skills (candidate_id UUID FK->candidates, skill VARCHAR) -- skills stored directly as text
position_skills (position_id UUID FK->positions, skill VARCHAR) -- skills stored directly as text

candidate_languages (id SERIAL PK, candidate_id UUID FK->candidates, language VARCHAR)

experience (id SERIAL PK, candidate_id UUID FK->candidates, title VARCHAR, company VARCHAR, location VARCHAR, start_date VARCHAR, end_date VARCHAR, description TEXT, sort_order INT)

education (id SERIAL PK, candidate_id UUID FK->candidates, degree VARCHAR, institution VARCHAR, start_date VARCHAR, end_date VARCHAR)

certifications (id SERIAL PK, candidate_id UUID FK->candidates, name VARCHAR, year INT)

candidate_positions (candidate_id UUID FK->candidates, position_id UUID FK->positions, created_at TIMESTAMPTZ) -- assignment junction

position_requirements (id SERIAL PK, position_id UUID FK->positions, item TEXT, type VARCHAR ['required','nice_to_have','responsibility'], sort_order INT)

documents (id SERIAL PK, entity_type VARCHAR, entity_id UUID, filename VARCHAR, file_type VARCHAR, stored_path VARCHAR, created_at TIMESTAMPTZ)

Important notes:
- Dates in experience/education are VARCHAR (format: 'YYYY-MM' or 'YYYY'), not DATE type
- Use ILIKE for case-insensitive text matching
- Skills are stored directly in candidate_skills.skill and position_skills.skill (no lookup table)
- Experience bullets are stored in experience.description as newline-separated text
- Position responsibilities are in position_requirements with type='responsibility'
- Join candidates to positions through candidate_positions junction table
- The users table exists but is NOT accessible (contains credentials)
- Use LOWER() for case-insensitive comparisons on indexed columns
